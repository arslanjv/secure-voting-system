About Secure Online Voting System

Overview

The Secure Online Voting System is a production-ready, enterprise-grade electronic voting platform built with security and transparency as the core principles. This system enables secure, auditable, and verifiable elections while maintaining voter privacy.

Key Features
End-to-End Encryption

All votes are encrypted using AES-256-GCM encryption on both client and server side. Votes remain encrypted at rest and are only decrypted during official tallying by administrators.

Digital Signatures

Ed25519 digital signatures ensure vote integrity and authenticity. Every vote and audit log entry is cryptographically signed for verification.

Two-Factor Authentication

TOTP-based 2FA adds an extra layer of security to user accounts, preventing unauthorized access even if passwords are compromised.

Voter Privacy

Vote verification tokens allow voters to verify their votes were counted correctly without revealing their actual choices to others.

Comprehensive Audit Logs

Blockchain-style audit logging with cryptographic chaining ensures tamper-evident record-keeping of all system activities.

Role-Based Access Control

Three distinct roles (Voter, Administrator, Auditor) ensure proper separation of duties and principle of least privilege.

Replay Attack Prevention

Cryptographic nonces prevent vote replay attacks. Each vote submission requires a unique, time-limited nonce.

Rate Limiting

Built-in rate limiting prevents brute force attacks and denial-of-service attempts, protecting system availability.

Technology Stack
  • Backend: Python 3.11 + Flask 3.0
  • Database: PostgreSQL with SQLAlchemy ORM
  • Encryption: PyCryptodome (AES-256-GCM), cryptography (Ed25519)
  • Authentication: Flask-Login, pyotp (TOTP 2FA), Argon2 password hashing
  • Security: Flask-Talisman (HSTS, CSP), Flask-Limiter (rate limiting)
  • Frontend: Bootstrap 5, vanilla JavaScript with Web Crypto API
System Architecture

The system follows a modular Blueprint-based architecture with clear separation of concerns:

  • Authentication Module: Handles user login, registration, and 2FA
  • Voter Module: Manages voting interface and vote verification
  • Admin Module: Controls election creation, management, and tallying
  • Auditor Module: Provides audit log access and verification tools
Open Source: This system is built on open-source technologies and follows industry best practices for security and transparency.
