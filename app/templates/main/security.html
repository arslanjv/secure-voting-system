Security Information

Your Security is Our Priority

This voting system implements multiple layers of security to protect your vote and personal information. Learn more about our security measures below.

Cryptographic Security
AES-256-GCM Encryption

What it does: Encrypts your vote so only authorized administrators can decrypt it during official tallying.

How it works: Uses 256-bit keys with Galois/Counter Mode for authenticated encryption.

Security level: Military-grade encryption, resistant to all known attacks.

Ed25519 Digital Signatures

What it does: Proves your vote hasn't been tampered with after submission.

How it works: Creates cryptographic signatures using elliptic curve cryptography.

Security level: 128-bit security equivalent, fast and secure.

Authentication Security
Argon2 Password Hashing

What it does: Protects your password even if the database is compromised.

How it works: Uses memory-hard hashing with 64MB memory cost and 3 iterations.

Security level: Winner of Password Hashing Competition, resistant to GPU/ASIC attacks.

TOTP Two-Factor Authentication

What it does: Adds a second layer of protection beyond your password.

How it works: Generates time-based one-time codes using HMAC-SHA1.

Security level: Protects against password theft and phishing attacks.

Application Security
  • CSRF Protection: Prevents cross-site request forgery attacks on all forms
  • SQL Injection Prevention: ORM-only queries, no raw SQL execution
  • XSS Protection: Input sanitization and output encoding on all user data
  • Rate Limiting: Prevents brute force and denial-of-service attacks
  • Account Lockout: Automatically locks accounts after 5 failed login attempts
  • Session Security: HttpOnly, Secure, SameSite cookies with 30-minute timeout
  • HTTPS Enforcement: TLS 1.3 only, HSTS enabled with 1-year duration
  • Content Security Policy: Prevents unauthorized script execution
Audit & Transparency
Blockchain-Style Audit Logs

Every action in the system is logged with cryptographic chaining. Each audit log entry contains:

  • SHA-256 hash of the previous entry (creating an immutable chain)
  • Timestamp and user information
  • Action performed and details
  • Digital signature for verification

Auditors can verify: Any tampering with audit logs will break the cryptographic chain, making it immediately detectable.

Vote Verification
Individual Verifiability

After voting, you receive a unique verification token. You can use this token to:

  • Verify your vote was recorded in the system
  • Check your vote hasn't been altered
  • Confirm your vote will be counted in the final tally

Privacy protected: The verification system doesn't reveal your vote choice, only that your encrypted vote exists in the database.

Security Best Practices
How You Can Stay Secure:
  • Use strong passwords: At least 12 characters with uppercase, lowercase, numbers, and symbols
  • Enable 2FA: Always enable two-factor authentication for your account
  • Keep credentials private: Never share your password or 2FA codes
  • Log out after voting: Always log out when finished, especially on shared computers
  • Verify SSL/TLS: Check for the padlock icon in your browser before entering credentials
  • Save your verification token: Keep your vote verification token secure but accessible
  • Report suspicious activity: Contact administrators if you notice anything unusual
